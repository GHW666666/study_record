## 1. 多环境 × 多形态构建体系（核心设计）

工程最核心的设计是**两个维度正交组合**，共 6 种构建模式。

**类型定义（L21-23）** —— 用 TS 模板字面量类型约束出 6 种合法组合：

```ts
type BaseMode = 'dev' | 'test' | 'prod';
type SubMode = 'offline' | 'online';
type AppMode = `${BaseMode}-${SubMode}`; // dev-offline | dev-online | test-offline ...
```

**模式解析（L47-65）** —— 把 `--mode xxx` 拆成 5 个开关，驱动后续所有差异化行为：

```ts
const parseMode = (mode: string) => {
    const [baseMode, subMode] = mode.split('-') as [BaseMode, SubMode];
    const resolvedBaseMode = baseMode || 'dev';
    return {
        baseMode: resolvedBaseMode,
        subMode: subMode || 'online',
        isOffline: subMode === 'offline',
        isTest: resolvedBaseMode === 'test',
        isProd: resolvedBaseMode === 'prod',
    };
};
```

**环境变量加载 + base 路径（L139、L144）** —— `loadEnv` 按 mode 名自动读对应 `.env.xxx`；`base` 决定资源前缀（online 走 CDN，offline 走本地 `/`）：

```ts
const env = loadEnv(mode as AppMode, process.cwd(), 'VITE_');
// ...
base: env.VITE_BASE_PATH || '/',
```

## 8. 构建优化（build，L372-399）

**sourcemap 仅 test 生成（L375）** —— prod 不生成防源码泄露：

```ts
sourcemap: isTest,
chunkSizeWarningLimit: isProd ? 800 : 1000,
```

**代码分割 manualChunks（L381-391）** —— 按依赖 + 页面维度拆包，实现按需加载：

```ts
manualChunks(id) {
    if (id.includes('node_modules')) {
        if (id.includes('vue') || id.includes('vue-router')) {
            return 'vendor-vue';       // vue 全家桶单独一包
        }
        return 'vendor-other';         // 其他三方库
    }
    if (id.includes('src/views')) {
        return `page-${id.split('src/views/')[1].split('/')[0]}`; // 每个页面单独一包
    }
},
```

**hash 文件名（L392-394）** —— 内容变更才失效缓存：

```ts
entryFileNames: `${packageJson.name}/[name].[hash:8].js`,
chunkFileNames: `${packageJson.name}/[name].[hash:8].js`,
assetFileNames: `${packageJson.name}/[name].[hash:8].[ext]`,
```

## 工程化能力总结

- **多环境构建**：`dev/test/prod × offline/online` 六模式，`parseMode` + `loadEnv` 驱动（L47/L139）
- **低版本兼容**：`legacy` 插件 + `autoprefixer`，支持 iOS13 / Android5.1（L161/L249）
- **离线包机制**：`manifest` 生成 + `zip` 打包 + BOS 上传 + CSS 内联（L206-231）
- **性能优化**：按页面 / 依赖 `manualChunks`、hash 缓存、CSS 合并、chunk 体积告警（L374-394）
- **开发体验**：`proxy` 跨域、`0.0.0.0` 真机调试、HMR、SCSS 全局 mixin（L294-370）
- **可追溯性**：版本号 + 时间戳、构建模式全局变量注入、test sourcemap（L43/L171/L375）
- **质量保障**：`vitest` + `jsdom` + 覆盖率报告（L281-292）
