# TS/JS 类型系统学习笔记

> 整理自实际项目（nd-aichat-app）代码阅读中遇到的类型问题，从 JS 运行时类型判断讲到 TS 类型层的高级用法。

---

## 一、核心心智模型：两个世界

理解 TS 类型系统的第一步，是分清代码活在哪个世界：

| 世界 | 何时存在 | 代表语法 |
|---|---|---|
| **运行时（JS）** | 代码真正执行时 | `typeof x`（值位置）、`instanceof` |
| **类型层（TS）** | 只在编译检查时，编译后**全部消失** | `typeof x`（类型位置）、`InstanceType<T>`、泛型、工具类型 |

**判断口诀：这段代码编译成 JS 之后还在不在？**

- 还在（写在 `if`、赋值等"值的位置"）→ 运行时判断
- 不在（写在冒号后、`type` 里、泛型尖括号里）→ 类型层操作

---

## 二、运行时的类型判断（JS）

### 2.1 `typeof`（值位置）

返回一个**字符串**，用于判断基本类型：

```javascript
typeof 'abc'      // 'string'
typeof 123        // 'number'
typeof undefined  // 'undefined'
typeof {}         // 'object'
typeof []         // 'object'（注意：数组也是 'object'）
typeof null       // 'object'（历史遗留 bug，记住即可）
```

典型场景：

```javascript
if (typeof value === 'string') {
    // 这里 TS 也会自动把 value 收窄为 string 类型
}
```

### 2.2 `instanceof`

返回一个**布尔值**，判断"这个对象是不是某个类（或其子类）new 出来的"：

```javascript
class Dog {}
const d = new Dog();

d instanceof Dog     // true
[] instanceof Array  // true
new TypeError('x') instanceof Error  // true（子类也算）
```

典型场景：区分错误类型

```javascript
try { /* ... */ } catch (e) {
    if (e instanceof TypeError) { /* ... */ }
}
```

## 三、类型层的 `typeof`（TS 类型查询）

**同一个关键字，位置不同含义完全不同。** 当 `typeof` 出现在类型位置（冒号后、泛型里），它的意思是"把这个变量的类型抄过来"：

```typescript
const config = { port: 3000, host: 'localhost' };

type Config = typeof config;  // { port: number; host: string }
let other: typeof config;     // other 的类型和 config 一样
```

两个 `typeof` 对比：

```typescript
if (typeof x === 'string') {}   // 值位置 → JS 运行时，返回字符串
let y: typeof x;                // 冒号后 → TS 类型层，复制类型
```

**为什么需要它？** 因为泛型尖括号这种"类型位置"只能写类型，不能写值。import 进来的组件、定义好的函数、普通对象都是"值"，要在类型位置使用它们，必须先用 `typeof` 把类型取出来。

## 四、实战案例：Vue ref 的泛型

### 4.1 `ref<Record<string, unknown> | null>(null)`

```typescript
const treeSummaryNode = ref<Record<string, unknown> | null>(null);
```

三层拆解：

1. **`ref<T>(初始值)`** —— 泛型 T 声明 `.value` 能装什么类型。不写的话 TS 会从初始值 `null` 推断成 `Ref<null>`，以后赋对象会报错。
2. **`Record<string, unknown>`** —— "键是任意字符串、值是任意类型的对象"，等价于 `{ [key: string]: unknown }`。用于"结构复杂/没有现成类型定义"的对象。
   - 值用 `unknown` 而不用 `any`：`unknown` 取出后必须先做类型判断才能用，更安全；`any` 完全绕过检查。
3. **`| null`** —— 联合类型，允许值为 null。因为初始值就是 null（数据还没到），不加就通不过类型检查。

连读：**声明一个响应式变量，值要么是任意结构的对象、要么是 null，初始为 null。**

### 4.2 `ref<InstanceType<typeof AgentContainer> | null>(null)`

```typescript
const agentContainerRef = ref<InstanceType<typeof AgentContainer> | null>(null);
```

这是**组件模板引用（template ref）**的固定公式。两层嵌套拆解：

1. **`typeof AgentContainer`** —— AgentContainer 是 import 进来的组件对象（一个值），用类型层 `typeof` 取出它的类型，得到"组件构造器"的类型。
2. **`InstanceType<...>`** —— 输入"类/构造器的类型"，输出"new 出来的实例的类型"。

类比普通 class：

```typescript
class Foo {
    bar() {}
}

type A = typeof Foo;        // "Foo 这个类本身"的类型
type B = InstanceType<A>;   // Foo 实例的类型，等于我们平时写的 Foo
```

**为什么组件要绕这个圈？** 类名写在类型位置时默认就是实例类型（`const a: Dog`），但 Vue 组件 import 进来的只是组件定义这个"值"，没有现成的实例类型名，只能用 `InstanceType<typeof 组件>` 推导。拿到实例类型后，`defineExpose` 暴露的属性方法才有补全和检查。

对照记忆：

- `ref<HTMLElement | null>(null)` —— 引用普通 DOM 元素
- `ref<InstanceType<typeof 某组件> | null>(null)` —— 引用子组件实例

### 4.3 四种"typeof/instanceof 家族"总对照

| 写法 | 世界 | 输入 → 输出 | 典型场景 |
|---|---|---|---|
| `typeof x`（值位置） | JS 运行时 | 值 → 字符串 | `if (typeof x === 'string')` |
| `x instanceof Foo` | JS 运行时 | 对象+类 → 布尔 | `e instanceof TypeError` |
| `typeof x`（类型位置） | TS 类型层 | 变量 → 它的类型 | `type C = typeof config` |
| `InstanceType<T>` | TS 类型层 | 类的类型 → 实例类型 | `InstanceType<typeof AgentContainer>` |

## 五、TS 内置工具类型（Utility Types）

工具类型 = 类型层面的函数：输入已有类型，加工出新类型。掌握下面这些能覆盖日常 90% 场景。

统一用这个类型做例子：

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}
```

### 5.1 改对象字段

**`Partial<T>` —— 全部变可选**

```typescript
type UserDraft = Partial<User>;
// { id?: number; name?: string; email?: string }

// 典型场景：更新函数只传要改的字段
function updateUser(id: number, changes: Partial<User>) {}
updateUser(1, { name: '新名字' });
```

**`Required<T>` —— 全部变必填**（Partial 的反操作）

```typescript
type FullUser = Required<UserDraft>;
```

**`Pick<T, K>` —— 挑几个字段**

```typescript
type UserPreview = Pick<User, 'id' | 'name'>;
// { id: number; name: string }
```

**`Omit<T, K>` —— 去掉几个字段**（Pick 的反操作）

```typescript
type CreateUserParams = Omit<User, 'id'>;  // 创建时还没有 id
```

**`Record<K, V>` —— 构造对象类型**

```typescript
type UserMap = Record<string, User>;
type Scores = Record<'math' | 'english', number>;
// { math: number; english: number }
```

**`Readonly<T>` —— 全部字段变只读**

```typescript
type FrozenUser = Readonly<User>;  // 字段不可再赋值
```

### 5.2 改联合类型

```typescript
type Status = 'pending' | 'success' | 'error';
```

**`Exclude<T, U>` —— 排除**

```typescript
type DoneStatus = Exclude<Status, 'pending'>;  // 'success' | 'error'
```

**`Extract<T, U>` —— 提取**

```typescript
type OkStatus = Extract<Status, 'success'>;  // 'success'
```

**`NonNullable<T>` —— 去掉 null 和 undefined**

```typescript
type MaybeUser = User | null | undefined;
type DefiniteUser = NonNullable<MaybeUser>;  // User
```

### 5.3 从函数/类推导

这一组常和类型层 `typeof` 组合使用（固定搭配）。

**`ReturnType<T>` —— 取函数返回值类型**

```typescript
function getUser() {
    return { id: 1, name: 'a', email: 'b' };
}
type U = ReturnType<typeof getUser>;
```

**`Parameters<T>` —— 取函数参数类型（元组）**

```typescript
type Args = Parameters<typeof updateUser>;
// [id: number, changes: Partial<User>]
```

**`Awaited<T>` —— 取 Promise 解开后的类型**

```typescript
async function fetchUser(): Promise<User> { /* ... */ }
type FetchedUser = Awaited<ReturnType<typeof fetchUser>>;  // User
```

**`InstanceType<T>` —— 取构造器的实例类型**（见第四节）

### 5.4 记忆框架

按"输入什么、干什么"分三组：

| 分组 | 工具类型 | 一句话 |
|---|---|---|
| 改对象字段 | `Partial` / `Required` / `Pick` / `Omit` / `Record` / `Readonly` | 变可选 / 变必填 / 挑 / 删 / 造 / 冻结 |
| 改联合类型 | `Exclude` / `Extract` / `NonNullable` | 排除 / 提取 / 去空 |
| 从函数/类推导 | `ReturnType` / `Parameters` / `Awaited` / `InstanceType` | 返回值 / 参数 / Promise 内值 / 实例 |

工具类型可以随意嵌套组合：

```typescript
type X = Partial<Omit<User, 'id'>>;  // 去掉 id 后全变可选
```

## 六、工具类型没有魔法

所有工具类型都是用三个底层特性实现的：`keyof`（取键）、`in`（映射类型）、`extends ? :`（条件类型）。例如 `Partial` 的源码只有一行：

```typescript
type Partial<T> = { [P in keyof T]?: T[P] };
// 遍历 T 的每个键 P，类型原样保留，但加 ? 变可选
```

**下一步学习方向**：`keyof`、映射类型（Mapped Types）、条件类型（Conditional Types）、`infer` —— 这些是自己写工具类型的入口，也是真正的"TS 高级用法"。
